      Machina Namespace

       

      Classes

      Action Actions represent high-level abstract operations such as movements, rotations, transformations or joint manipulations, both in absolute and relative terms. They are independent from the device's properties, and their translation into actual robotic instructions depends on the robot's properties and state.
      ActionAcceleration 
      ActionArcMotion An Action representing an arc motion through two frames.
      ActionArmAngle An action to change the arm-angle value for 7-dof robotic arms.
      ActionAttachTool Attaches a Tool to the robot flange. Must have beeb previously defined on the Robot. If the robot already had a tool, it will be replaced by this one.
      ActionAxes An Action representing the raw angular values of the device's joint rotations.
      ActionBuffer A class that manages a FIFO list of Actions.
      ActionComment Adds a line comment to the compiled code
      ActionCoordinates An Action to change current Reference Coordinate System.
      ActionCustomCode Insert a line of custom code in the program.
      ActionDefineTool Defines a new Tool on the Robot that will be available for Attach/Detach
      ActionDetachTool Detaches any tool currently attached to the robot.
      ActionExecutedEventArgs Arguments for ActionExecuted events.
      ActionExternalAxis Set the vlaue of an external axis to the device.
      ActionExtrusion Turns extrusion on/off in 3D printers.
      ActionExtrusionRate Sets the extrusion rate in 3D printers in mm of filament per mm of lineal travel.
      ActionInitialization Apply general initialization parameters to the device.
      ActionIOAnalog Writes a value to analog pin #.
      ActionIODigital Turns digital pin # on or off.
      ActionIssuedEventArgs Arguments for ActionIssued events.
      ActionMessage An Action representing a string message sent to the device to be displayed.
      ActionMotionMode An Action to change current MotionType.
      ActionPrecision An Action to change current precision settings.
      ActionPushPop An Action to Push or Pop current device settings (such as speed, precision, etc.)
      ActionReleasedEventArgs Arguments for ActionReleased events.
      ActionRotation An Action representing a Rotation transformation in Quaternion represnetation.
      ActionSpeed An Action to change the current speed setting.
      ActionTemperature Sets the temperature of the 3D printer part, and optionally waits for the part to reach the temp to resume eexecution.
      ActionTransformation An Action representing a combined Translation and Rotation Transformation.
      ActionTranslation An action representing a Translation transform in along a guiding vector.
      ActionWait An Action represening the device staying idle for a period of time.
      Compiler An abstract class that features methods to translate high-level robot actions into platform-specific programs.
      CompilerABB A compiler for ABB 6-axis industrial robotic arms.
      CompilerHuman A quick compiler for human-readable instructions.
      CompilerKUKA A compiler for KUKA 6-axis industrial robotic arms.
      CompilerMACHINA A simple compiler that returns a program in native Machina language, i.e. serializing each Action into its own Instruction form.
      CompilerUR A compiler for Universal Robots 6-axis robotic arms.
      CompilerZMORPH A compiler for ZMorph 3D printers.
      Control The core class that centralizes all private control.
      IO 
      IssueActionManager This class holds Methods to issue Actions and then use a control class to apply the action request
      Logger A class to bind Machina logging information to custom outputs via events.
      LoggerArgs Custom logging arguments
      MachinaEventArgs 
      MotionUpdateEventArgs Arguments for MotionUpdate events.
      Robot The core Class in Machina. Represents a state and action-based virtual robot, and exposes the public API for robot manipulation and control.
      RobotCursor Represents an abstraction of the state of a robotic device. Keeps track of things such as position, orientation, joint configuration, current speed, zone, etc. Useful as virtual representation of a simulated or controlled robot actuator.
      RobotLogger A "Console" class that can be attached to objects to track their log messages.
      Settings A class representing a Setting state, to be un/buffered.
      SettingsBuffer A buffer manager for Settings objects.
      Tool Represents a tool object that can be attached to the end effector of the robot. This class is public and will be used directly by the user, so careful design of the API vs. internal methods will be relevant.

      Enumerations

      ActionType Defines an Action Type, like Translation, Rotation, Wait... Useful to flag base Actions into children classes.
      ConnectionType Defines who will be in charge of setting up a device for correct connection, i.e. having Machina try to load a server/firmata modules to the controller or leave that task to the User (default).
      ControlType Represents the type of control that will be performed over the real/virtual robot.
      CursorType Defines what state is being represented by a cursor.
      CycleType Defines the different cycle type modes a program can be ran.
      ExecutionState 
      ExternalAxesTarget Defines where will the external axes be applied to
      ExternalAxisType Defines the types of External Axes
      LogLevel Defines how verbose should internal logging be
      MotionType Defines which type of motion to use for translation actions.
      ReferenceCS Defines which reference coordinate system to use for transform actions.
